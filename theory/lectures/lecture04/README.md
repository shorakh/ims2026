## 1. Цели и виды преобразования информации

**Преобразование информации** — изменение формы представления без изменения содержания.

| Цель преобразования | Метод | Примеры |
|--------------------|-------|---------|
| Уменьшить объём | Сжатие | ZIP, MP3, JPEG |
| Защитить от ошибок | Помехозащитное кодирование | Код Хэмминга, CRC |
| Защитить от доступа | Шифрование | AES, RSA |
| Передать по каналу | Модуляция | AM, FM, QAM |
| Унифицировать | Стандартное кодирование | ASCII, UTF-8, Unicode |

## 2. Коды и кодирование

**Код** — правило взаимно однозначного отображения символов алфавита в кодовые слова.

### Равномерный код
Все кодовые слова имеют одинаковую длину $l = \lceil \log_2 n \rceil$.

Пример: ASCII — 8 бит на символ, независимо от частоты.

### Неравномерный (переменной длины) код
Частые символы кодируются **короткими** словами, редкие — **длинными**.

**Условие Крафта** — необходимое и достаточное условие существования однозначно декодируемого кода:
$$\sum_{i=1}^{n} 2^{-l_i} \leq 1$$

**Префиксный код** — ни одно кодовое слово не является префиксом другого. Такой код всегда однозначно декодируется.

## 3. Код Хаффмана

### Алгоритм построения

1. Упорядочить символы по убыванию вероятности
2. Объединить два символа с наименьшими вероятностями в один узел
3. Повторять до одного узла (корня)
4. Пройти дерево от корня: левая ветвь = 0, правая = 1

### Пример

Символы: A(0.4), B(0.2), C(0.2), D(0.1), E(0.1)

```
Шаг 1: A(0.4) B(0.2) C(0.2) D(0.1) E(0.1)
                                     └─┬─┘
                                     DE(0.2)

Шаг 2: A(0.4) B(0.2) C(0.2) DE(0.2)
               └────────┬────────┘
                      BCD(0.4)   ← ошибка, пересчитаем...
```

**Дерево Хаффмана:**

```
              root(1.0)
             /          \
          A(0.4)       (0.6)
                       /    \
                    B(0.2)  (0.4)
                            /    \
                         C(0.2) (0.2)
                                /    \
                             D(0.1) E(0.1)
```

**Кодовая таблица:**

| Символ | p | Код | Длина |
|--------|---|-----|-------|
| A | 0.4 | 0 | 1 |
| B | 0.2 | 10 | 2 |
| C | 0.2 | 110 | 3 |
| D | 0.1 | 1110 | 4 |
| E | 0.1 | 1111 | 4 |

**Средняя длина кодового слова:**
$$\bar{L} = 0.4 \cdot 1 + 0.2 \cdot 2 + 0.2 \cdot 3 + 0.1 \cdot 4 + 0.1 \cdot 4 = 2.2 \text{ бит}$$

**Энтропия:** $H = 2.122$ бит. КПД: $\eta = 2.122 / 2.2 = 0.965 = 96.5\%$

## 4. Код Шеннона-Фано

### Алгоритм

1. Упорядочить символы по убыванию вероятности
2. Разбить на две группы с примерно равными суммами вероятностей
3. Левой группе назначить 0, правой — 1
4. Рекурсивно повторить для каждой группы

### Пример (те же символы)

```
A(0.4) B(0.2) C(0.2) D(0.1) E(0.1)
└──────0──────┘ └──────1──────────┘
               ↓               ↓
         A → 0            B C D E
         B(0.2) C(0.2)  B(0.2)C(0.2) D(0.1)E(0.1)
         └─0─┘ └─1─┘    └─0─┘ └─1─┘ └──0──┘└──1──┘
```

| Символ | Код | Длина |
|--------|-----|-------|
| A | 0 | 1 |
| B | 10 | 2 |
| C | 11 | 2 |
| D | 110 | ... |

*(Детальное построение зависит от разбиения)*

### Хаффман vs Шеннона-Фано

| Параметр | Хаффман | Шеннона-Фано |
|---------|---------|--------------|
| Оптимальность | ✅ Оптимальный | ❌ Субоптимальный |
| Направление | Снизу вверх | Сверху вниз |
| Сложность | $O(n \log n)$ | $O(n \log n)$ |

## 5. Словарные методы сжатия (LZ-семейство)

### LZ77 (Lempel-Ziv 1977)

Использует **скользящее окно**: найти в предыдущем тексте самое длинное совпадение.

Кодовое слово: $(offset, length, next\_char)$

```
Текст: "абракадабра"
       ┌────────────── словарь (окно) ──┐
       а б р а к а д | а б р а
                       ↑
              найдено совпадение "абр" на позиции -7
              → (7, 3, 'а')
```

### LZ78 и LZW

LZW строит словарь в процессе кодирования:
- Стартовый словарь: все одиночные символы
- При каждом шаге: найти самую длинную строку в словаре, вывести её код, добавить расширенную строку в словарь

**LZW используется в:** GIF, TIFF, PDF (частично), ZIP (вариант).

## 6. Сжатие с потерями

Применяется для мультимедиа, где допустима небольшая потеря качества.

| Формат | Применение | Метод |
|--------|-----------|-------|
| JPEG | Фото | DCT + квантование + Хаффман |
| MP3 | Аудио | Психоакустическая модель + Хаффман |
| H.264/H.265 | Видео | Межкадровое предсказание + DCT |

**Коэффициент сжатия JPEG:** 10:1–20:1 при хорошем качестве.

## 7. Помехозащитное кодирование

Добавление **избыточности** для обнаружения и исправления ошибок.

### Код с битом чётности

Добавляем 1 бит так, чтобы сумма всех битов была чётной (или нечётной).

```
Данные:    1 0 1 1
Бит чётн.: 1       (XOR всех битов)
Кодовое:   1 0 1 1 1
```
Обнаруживает 1 ошибку, исправить не может.

### Код Хэмминга (7,4)

Кодирует **4** информационных бита в **7** кодовых. Исправляет 1 ошибку.

Позиции контрольных битов: 1, 2, 4 (степени двойки).

```
Позиции: 1  2  3  4  5  6  7
Биты:    p1 p2 d1 p3 d2 d3 d4
```

**Синдром ошибки** — XOR контрольных битов — указывает позицию ошибочного бита.

### Соотношение для кода Хэмминга

$$2^r \geq m + r + 1$$

где $r$ — число контрольных битов, $m$ — число информационных.

| m | r | n = m+r | Скорость R = m/n |
|---|---|---------|-----------------|
| 4 | 3 | 7 | 0.571 |
| 11 | 4 | 15 | 0.733 |
| 26 | 5 | 31 | 0.839 |

### Задача

> Постройте код Хаффмана для символов: X(0.5), Y(0.3), Z(0.2). Вычислите $\bar{L}$ и $H$.

<details>
<summary>Решение</summary>

Дерево: объединяем Y(0.3) и Z(0.2) → YZ(0.5), затем X(0.5) и YZ(0.5) → root(1.0)

| Символ | Код |
|--------|-----|
| X | 0 |
| Y | 10 |
| Z | 11 |

$\bar{L} = 0.5 \cdot 1 + 0.3 \cdot 2 + 0.2 \cdot 2 = 0.5 + 0.6 + 0.4 = 1.5$ бит

$H = -(0.5 \log_2 0.5 + 0.3 \log_2 0.3 + 0.2 \log_2 0.2) = 0.5 + 0.521 + 0.464 = 1.485$ бит

КПД: $\eta = 1.485/1.5 = 99\%$ — почти оптимально!
