## 1. Понятие обработки информации

**Обработка информации** — целенаправленный процесс изменения содержания или формы представления информации.

Различают:
- **Обработку данных** — преобразование по формальным правилам (сортировка, поиск)
- **Переработку информации** — получение новых знаний (анализ, синтез)

### Типовые операции обработки данных

| Операция | Описание |
|----------|----------|
| **Сбор** | Получение данных из источников |
| **Регистрация** | Первичная фиксация |
| **Сортировка** | Упорядочивание по ключу |
| **Поиск** | Нахождение по критерию |
| **Агрегирование** | Обобщение (сумма, среднее) |
| **Преобразование** | Изменение формата |
| **Выдача** | Представление пользователю |

### Технологическая сеть обработки данных

```
Источник → Ввод → Обработка → Хранение → Вывод → Пользователь
              ↑         ↑          ↑
           ОС, СУБД, ПО обработки
```

## 2. Физические основы хранения информации

### Типы носителей

| Тип | Принцип | Примеры | Плотность |
|-----|---------|---------|-----------|
| Магнитный | Намагниченность | HDD, магнитная лента | до 1 Тб/дюйм² |
| Оптический | Отражение лазера | CD, DVD, Blu-ray | до 25 Гб/слой |
| Флэш (SSD) | Заряд в ячейке | USB, SSD, SD-карта | до 10+ Тб на чип |
| Оперативная | Конденсатор/триггер | DRAM, SRAM | быстрое, энергозависимое |

### Иерархия памяти

```
     ┌─────────────┐  быстро, дорого, мало
     │  Регистры   │  ~1 нс
     ├─────────────┤
     │   Кэш L1   │  ~2 нс
     ├─────────────┤
     │   Кэш L2   │  ~5 нс
     ├─────────────┤
     │     RAM     │  ~100 нс
     ├─────────────┤
     │     SSD     │  ~0.1 мс
     ├─────────────┤
     │     HDD     │  ~10 мс
     └─────────────┘  медленно, дёшево, много
```

## 3. Базы данных и хранилища данных

**База данных (БД)** — организованная совокупность структурированных данных.

### Типы СУБД

| Модель | Описание | Примеры |
|--------|----------|---------|
| Реляционная | Таблицы, SQL | MySQL, PostgreSQL, Oracle |
| Документная | JSON-документы | MongoDB, CouchDB |
| Ключ-значение | Простые пары | Redis, Memcached |
| Графовая | Узлы и рёбра | Neo4j |
| Колоночная | По столбцам | Cassandra, ClickHouse |

**Хранилище данных (Data Warehouse)** — централизованное хранилище исторических данных для аналитики (OLAP).

**Технология RAID** — Redundant Array of Independent Disks:
- **RAID 0** — Чередование (скорость, нет защиты)
- **RAID 1** — Зеркало (надёжность, 50% ёмкости)
- **RAID 5** — Чередование + чётность (баланс)
