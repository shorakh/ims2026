## 1. Концепция открытых систем

**Открытая система** — система, реализованная на основе стандартов, которые открыты (опубликованы и доступны), обеспечивая совместимость с другими системами.

### Ключевые принципы

| Принцип | Описание |
|---------|----------|
| **Открытость** | Стандарты доступны всем |
| **Переносимость** | Работает на разных платформах |
| **Интероперабельность** | Взаимодействует с другими системами |
| **Масштабируемость** | Наращивание без переработки |
| **Дружелюбность** | Единый интерфейс пользователя |

### Документы открытых систем

- **OSI** (Open Systems Interconnection) — эталонная модель ISO
- **POSIX** — переносимость приложений Unix
- **W3C** — стандарты веба (HTML, CSS, XML)

## 2. Эталонная модель OSI

**OSI (Open Systems Interconnection)** — 7-уровневая концептуальная модель сетевого взаимодействия.

```
  Передатчик                              Приёмник
┌─────────────────┐                  ┌─────────────────┐
│ 7. Прикладной   │◄── данные ──────▶│ 7. Прикладной   │
│ 6. Представлен. │                  │ 6. Представлен. │
│ 5. Сеансовый    │                  │ 5. Сеансовый    │
│ 4. Транспортный │                  │ 4. Транспортный │
│ 3. Сетевой      │                  │ 3. Сетевой      │
│ 2. Канальный    │                  │ 2. Канальный    │
│ 1. Физический   │◄── биты ────────▶│ 1. Физический   │
└─────────────────┘                  └─────────────────┘
```

### Функции уровней OSI

| № | Уровень | Функция | Протоколы | Единица |
|---|---------|---------|-----------|---------|
| 7 | **Прикладной** | Интерфейс приложений | HTTP, FTP, SMTP, DNS | Данные |
| 6 | **Представления** | Кодирование, шифрование, сжатие | SSL/TLS, ASCII | Данные |
| 5 | **Сеансовый** | Управление сеансом | NetBIOS, RPC | Данные |
| 4 | **Транспортный** | Надёжная доставка | TCP, UDP | Сегмент |
| 3 | **Сетевой** | Маршрутизация | IP, ICMP, RIP, OSPF | Пакет |
| 2 | **Канальный** | Передача кадров | Ethernet, Wi-Fi, PPP | Кадр |
| 1 | **Физический** | Биты → сигналы | RS-232, USB, оптика | Бит |

### Инкапсуляция данных

```
Приложение: [Данные]
Транспорт:  [TCP-заголовок | Данные]
Сеть:       [IP-заголовок | TCP | Данные]
Канал:      [MAC-заголовок | IP | TCP | Данные | CRC]
Физический: биты
```

## 3. Стек протоколов TCP/IP

На практике используется 4-уровневая модель TCP/IP:

| TCP/IP | OSI уровни | Примеры |
|--------|-----------|---------|
| Прикладной | 5, 6, 7 | HTTP, FTP, DNS, SMTP |
| Транспортный | 4 | TCP, UDP |
| Сетевой | 3 | IP, ICMP |
| Сетевого доступа | 1, 2 | Ethernet, Wi-Fi |

### TCP vs UDP

| Параметр | TCP | UDP |
|----------|-----|-----|
| Надёжность | Гарантированная доставка | Нет гарантий |
| Порядок | Сохраняется | Не гарантирован |
| Скорость | Медленнее | Быстрее |
| Применение | HTTP, FTP, почта | Видеостриминг, DNS, игры |

## 4. IP-адресация

**IPv4**: 32 бита, запись: `192.168.1.1`  
**IPv6**: 128 бит, запись: `2001:0db8:85a3::8a2e:0370:7334`

### Классы IPv4 (устаревшая, но важная концепция)

| Класс | Диапазон | Число сетей | Хостов в сети |
|-------|---------|------------|--------------|
| A | 0.0.0.0 – 127.255.255.255 | 128 | 16 млн |
| B | 128.0.0.0 – 191.255.255.255 | 16 384 | 65 534 |
| C | 192.0.0.0 – 223.255.255.255 | 2 млн | 254 |

**Приватные диапазоны:** 10.x.x.x, 172.16–31.x.x, 192.168.x.x
