## 1. Технологии и средства поиска информации

### Принципы работы поисковых систем

```
Интернет → Краулер (паук) → Индекс → Ранжирование → Результаты
```

1. **Краулинг** — автоматический обход страниц
2. **Индексирование** — создание инвертированного индекса
3. **Ранжирование** — определение релевантности (PageRank и др.)

### Инвертированный индекс

```
"алгоритм" → [doc1, doc5, doc12, ...]
"шеннон"   → [doc2, doc5, doc8, ...]
```

### Меры релевантности

**TF-IDF** (Term Frequency — Inverse Document Frequency):
$$\text{TF-IDF}(t,d) = \text{TF}(t,d) \cdot \log\frac{N}{df(t)}$$

где TF — частота слова в документе, $N$ — число документов, $df(t)$ — число документов со словом $t$.

## 2. Мультимедиа технологии

**Мультимедиа** — интеграция нескольких видов информации: текст, графика, звук, видео, анимация.

### Форматы и кодеки

| Тип | Формат | Сжатие | Потери |
|-----|--------|--------|--------|
| Изображение | JPEG | DCT | Да |
| Изображение | PNG | Deflate | Нет |
| Изображение | WebP | Гибридный | Оба |
| Аудио | MP3 | Психоакустика | Да |
| Аудио | FLAC | LPC | Нет |
| Видео | H.264 | Межкадровое | Да |
| Видео | H.265 | Улучшенное | Да |

### Методы сжатия изображений

**JPEG (Joint Photographic Experts Group):**

```
Изображение → DCT → Квантование → Зигзаг-развёртка → Хаффман → Файл
```

1. **DCT** (Дискретное косинусное преобразование) — перевод в частотную область
2. **Квантование** — округление коэффициентов (здесь теряется информация)
3. **Энтропийное кодирование** — Хаффман

## 3. Криптографические алгоритмы

### Классификация шифров

```
Шифры
├── Симметричные (один ключ)
│   ├── Поточные (RC4, ChaCha20)
│   └── Блочные (AES, DES, 3DES)
└── Асимметричные (пара ключей)
    └── RSA, ECC, ElGamal
```

### Классические шифры

#### Шифр Цезаря
Сдвиг каждой буквы на $k$ позиций:
$$c_i = (m_i + k) \bmod n$$

Пример ($k=3$): A→D, B→E, Z→C

#### Шифр Виженера
Полиалфавитный шифр с ключевым словом:
$$c_i = (m_i + k_{i \bmod L}) \bmod 26$$

где $L$ — длина ключа.

#### XOR-шифр
$$c_i = m_i \oplus k_{i \bmod L}$$

При длине ключа = длине сообщения и случайном ключе = **одноразовый блокнот** (абсолютно стойкий).

---

### Современные шифры

#### AES (Advanced Encryption Standard)
- Блочный шифр, блок 128 бит
- Ключи: 128, 192 или 256 бит
- Режимы: ECB, CBC, CTR, GCM
- Используется везде: HTTPS, WiFi WPA2, ZIP

#### RSA
Основан на сложности факторизации больших чисел.

```
Генерация ключей:
1. Выбрать простые p, q
2. n = p·q
3. φ(n) = (p-1)(q-1)
4. Выбрать e: gcd(e, φ(n)) = 1
5. d = e⁻¹ mod φ(n)
Открытый ключ: (e, n)
Закрытый ключ: (d, n)

Шифрование: c = m^e mod n
Дешифрование: m = c^d mod n
```

### 4. Хэш-функции

**Хэш-функция** — однонаправленная функция, отображающая данные любого размера в строку фиксированной длины.

| Алгоритм | Размер хэша | Применение |
|----------|------------|------------|
| MD5 | 128 бит | Устарел (уязвим), контрольные суммы |
| SHA-1 | 160 бит | Устарел для безопасности |
| SHA-256 | 256 бит | Цифровые подписи, Bitcoin |
| SHA-3 | 224–512 бит | Современный стандарт |
| BLAKE3 | 256 бит | Быстрейший, современный |

**Свойства хэш-функции:**
1. Одностороннесть (необратимость)
2. Устойчивость к коллизиям (разные входы → разные хэши)
3. Лавинный эффект (1 бит на входе → ~50% бит изменяется)

### 5. Криптоанализ (основы для ЛР №6)

**Частотный анализ** — атака на моноалфавитные шифры.

Частоты букв русского языка (топ-10):
| Буква | Частота |
|-------|---------|
| о | 10.97% |
| е/ё | 8.45% |
| а | 8.01% |
| и | 7.35% |
| н | 6.70% |
| т | 6.26% |
| с | 5.47% |
| р | 4.73% |
| в | 4.54% |
| л | 4.34% |

**Индекс совпадений:**
$$IC = \frac{\sum_{i=0}^{n-1} f_i(f_i-1)}{N(N-1)}$$

- Русский: $IC \approx 0.055$
- Английский: $IC \approx 0.065$
- Случайный текст: $IC \approx 0.038$
